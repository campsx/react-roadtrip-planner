# Architecture — Feature-Based / Vertical Slice

## Path alias

Vite resolves `@` → `src/`. Always use `@/` for imports from `src/`.

```ts
import { client } from '@/lib/client';
import type { Country } from '@/features/countries/model/countries.types';
```

## App bootstrap (`src/app/`)

The application envelope lives in `src/app/` — nothing else belongs at `src/` root except `main.tsx` and `index.css`.

```
src/app/
├── App.tsx          # Root component — renders RouterProvider
├── router.tsx       # createBrowserRouter with all routes
├── providers.tsx    # AppProviders — aggregates all feature context providers
└── pages/           # Thin re-export proxies consumed by the router
    ├── Countries/page.tsx
    ├── CountryDetail/page.tsx
    ├── Login/page.tsx
    ├── NotFound/NotFound.tsx
    └── Roadtrip/page.tsx
```

> Providers that depend on router context (e.g. `AuthProvider` uses `useNavigate`) must stay **inside** the router tree — `AppProviders` is consumed by `RootLayout`, not by `App.tsx`.

## Feature anatomy

Each feature lives under `src/features/<name>/` with four sub-folders:

```
src/features/<name>/
├── api/
│   ├── index.ts          # Aggregates only — exports <name>Api object
│   ├── <action>.ts       # One file per API function
│   └── mappers.ts        # Raw API → domain type mapping (when shapes differ)
├── model/
│   ├── <name>.types.ts   # TypeScript interfaces/types for this feature
│   └── <name>.store.tsx  # React context + provider (the feature store)
├── hooks/
│   └── use<Name>.ts      # One file per hook — e.g. useAuth.ts, useRoadtrip.ts
└── components/
    └── <Name>Page.tsx    # Components and pages scoped to this feature
```

**Active features:** `auth` · `countries` · `roadtrip` · `notification`

> `notification` has no `api/` folder — features without remote data only need `model/` + `hooks/`.

## Generated files (`CLAUDE_ACTION/`)

Files generated by Claude commands (`/review`, `/refacto`, etc.) are written to `CLAUDE_ACTION/` at the project root. This folder is gitignored — never write generated reports or plans to the project root or anywhere else in the repo.

## Shared code (outside features)

| Path                       | Purpose                                         |
|----------------------------|-------------------------------------------------|
| `src/lib/client.ts`        | Axios instance (base URL, auth interceptors)    |
| `src/config/env.ts`        | Typed env variable accessors                    |
| `src/components/`          | Layout and shared UI primitives                 |
| `src/components/ui/`       | Generic UI atoms (ErrorMessage, Button…)        |
| `src/utils/`               | Pure stateless helpers (e.g. `debounce.ts`)     |
| `src/lib/`                 | Shared infrastructure too small for a feature  |

> Pages, router, and providers live in `src/app/` — see section above.

## Cross-feature imports

Use the public hooks from `hooks/` to read another feature's state.  
**Never** import directly from another feature's `store` or `api`.

```ts
// ✅ allowed
import { useAuth } from '@/features/auth/hooks/useAuth';

// ❌ forbidden
import { AuthContext } from '@/features/auth/model/auth.store';
```
